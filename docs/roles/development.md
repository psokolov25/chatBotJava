# Разработка

## 1. Архитектурные зоны
- `telegram` — state-machine, polling и рендеринг интерактивных сценариев.
- `queue` — REST-шлюз к Orchestra/Axioma.
- `events` + `cometd` + `kafka` — входящий поток событий вызова.
- `path` — клиентский путь и переходы по вопросам.
- `config`/`model`/`util` — конфигурация, доменные модели и утилиты.

## 2. Правила изменений
- Любая новая публичная API/логика должна иметь JavaDoc.
- Любой новый сценарий ветвления (branch/service selection) должен покрываться тестом.
- Логи не должны содержать PII в открытом виде.

## 3. Минимальный quality gate
- Unit-тесты для новой бизнес-логики.
- Проверка обратной совместимости конфигурации (`application.yml`, env).
- Smoke-тест ручного сценария `/start`.

## 4. Полезные команды
```bash
mvn -q test
mvn -q package
```
