# DevOps

## 1. Развёртывание
- Поддерживаемые режимы: Docker Compose / JVM deployment.
- Конфигурация: env vars + `application.yml`.
- Секреты: хранить во внешнем secret manager, не в репозитории.

## 2. Наблюдаемость
- Централизованный сбор логов (приложение + контейнер).
- Метрики и алерты:
  - недоступность `/api/bot/status`,
  - рост ошибок 4xx/5xx к upstream,
  - Kafka lag (для Axioma),
  - сбои CometD подписки (для Orchestra).

## 3. Эксплуатация
- Rolling restart с проверкой статуса после рестарта.
- Периодическая проверка сетевой связности и TLS-сертификатов.
- Проверка актуальности branch-конфига при добавлении новых филиалов.

## 4. Рекомендации по SRE-практикам
- Задать SLO на доступность bot API и долю успешных операций создания визита.
- Ввести error budget и правила инцидент-коммуникации.
- Поддерживать стандартный postmortem-шаблон.
