# Поддержка (кратко)

## Цель
За 2–5 минут понять: это локальная ошибка пользователя, сбой интеграции или системный инцидент.

## Быстрый алгоритм
1. Проверить статус сервиса: `curl http://localhost:8080/api/bot/status`.
2. Проверить свежие ошибки: `docker compose logs -f telegram-orchestra-bot`.
3. Повторить пользовательский путь: `/start` → филиал → услуга → талон.

## Если не работает
- **Нет услуг** → проверить queue API и branch-конфиг.
- **Нет талона** → проверить `entryPointId` и response upstream.
- **Нет уведомления о вызове** → проверить CometD/Kafka.

## Что сразу собрать для L2
- время (UTC), branchId, chatId (маскированно), фрагмент ошибки из логов.
